# 硬件事件驱动的多负载类型识别报告

## 1. 背景与目标

- **目标**：验证基于 eBPF + perf_event 收集的硬件性能计数（CPU 周期、指令数、缓存未命中、TLB 未命中等）是否能区分典型的五种负载类型：
  1. `cpu`（纯 CPU 计算型）
  2. `io`（I/O 密集型）
  3. `memory`（内存密集型）
  4. `mixed`（混合型）
  5. `prod_mixed`（生产型混合：Nginx + CPU 计算）
- **方法**：  
  1. 并行采集线程调度（sched_switch）与 CPU 核心性能事件，归因到具体线程  
  2. 合并各场景数据，构建完整数据集  
  3. 进行 EDA（箱线图、相关性热图）、PCA 可视化与简单分类器验证  

---

## 2. 数据概览

- **样本总量**：共 5 种负载、每种几千到上万条样本  
- **特征维度**：  
  - `cpu_cycles`、`instructions`、`branch_instrs`、`branch_misses`  
  - `cache_misses`、`L1_icache_miss`、`L1_dcache_miss`  
  - `ITLB_misses`、`L2_cache_miss`、`DTLB_misses`  

---

## 3. 探索性数据分析（EDA）

### 3.1 箱线图对比

![箱线图合集](results/monitor/boxplots.png)

- **CPU 周期**：`cpu` 与 `prod_mixed` 明显更高，`io` 最低  
- **缓存 & TLB 未命中**：不同场景分布差异显著  
  - `cpu` 场景下 L1 数据未命中极高  
  - `mixed` 和 `prod_mixed` 呈中等水平  
  - `memory`、`io` 则更低  

### 3.2 相关性热图

![相关性热图](results/monitor/corr_heatmap.png)

- **高度正相关**：  
  - `instructions ↔ branch_instrs`（0.83）  
  - `cache_misses ↔ L1_dcache_miss`（0.88）  
- **中度相关**：  
  - `cpu_cycles ↔ instructions`（0.75）  
  - `cpu_cycles ↔ L2_cache_miss`（0.34）  
- **弱或负相关**：可作为冗余特征裁剪或降维候选  

---

## 4. PCA 降维可视化

![PCA 散点图](results/monitor/pca_scatter.png)

- **前两主成分**已能将五种负载大体分离成 distinct clusters。  
- **重叠风险**：少数 `mixed` 与 `prod_mixed` 样本在特征空间相近，提示后续可引入更多 I/O/内存带宽相关特征。

---

## 5. 分类器验证

使用 **LogisticRegression**（默认参数）在 70% 训练 / 30% 测试划分下进行五类分类。

|             | Precision | Recall | F1-score |
|-------------|-----------|--------|----------|
| **cpu**     | 0.99      | 0.93   | 0.96     |
| **io**      | 0.94      | 0.89   | 0.91     |
| **memory**  | 0.88      | 0.94   | 0.91     |
| **mixed**   | 0.85      | 0.87   | 0.86     |
| **prod_mixed** | 0.89   | 0.95   | 0.92     |
| **accuracy**|           |        | **0.91** |

![混淆矩阵](results/monitor/confusion_matrix.png)

- **高识别率**：整体准确率约 91%  
- **主要混淆**：  
  - `memory ↔ io`（部分样本相互错判）  
  - `mixed ↔ prod_mixed`（少量交叉）  

---

## 6. 结论与展望

1. **硬件事件计数足够区分不同负载**，可作为在线“指纹”用于负载感知  
2. **特征间有冗余**，后续可通过 PCA / 特征选择简化模型  
3. **分类器效果良好**，简单模型已达 90%+ 准确率  
4. **下一步建议**  
   - 引入更多与 I/O 延迟、内存带宽、NUMA 拓扑相关的计数  
   - 尝试更强分类器（随机森林、XGBoost）优化混淆类别  
   - 将模型部署到调度/编排系统，实现实时负载感知与自适应调度
