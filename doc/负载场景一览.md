# 负载场景一览

## 1. 纯 CPU 计算型（cpu）

- **场景说明**：典型的矩阵乘法计算，纯计算密集

- **负载命令**：

  ```c
  stress-ng --cpu 8 --cpu-method matrixprod --timeout 300s
  ```

- **采集脚本**：`run_cpu.py`
- **输出目录**：`stress-results/cpu/`
  - `cpu.csv` — CPU 性能事件原始数据
  - `sched.csv` — 线程调度(sched_switch)事件日志
  - `attr.csv` — 归因后线程–事件对应表

------

## 2. 磁盘 I/O 场景（io）

- **场景说明**：顺序写 + 随机读写混合

- **负载命令**：

  ```
  stress-ng --hdd 4 --hdd-bytes 2G --timeout 300s
  fio --name=randrw --ioengine=libaio --direct=1 \
      --rw=randrw --rwmixread=70 --bs=4k --size=1G \
      --numjobs=4 --runtime=300 --time_based
  ```

**采集脚本**：`run_io.py`

**输出目录**：`stress-results/io/`

## 3.内存密集型（memory）

- **场景说明**：大块内存拷贝 + memtester 压力测试

- **负载命令**：

  ```
  stress-ng --vm 4 --vm-bytes 1G --vm-method memcpy --timeout 300s
  timeout 300s memtester 4G 5 --disable-progress
  ```

**采集脚本**：`run_mem.py`

**输出目录**：`stress-results/memory/`

## 4.综合混合负载（mixed）

- **场景说明**：CPU、I/O、内存同时施压

- **负载命令**：

  ```
  stress-ng --cpu 4 --io 4 --vm 2 --vm-bytes 2G --timeout 300s
  ```

**采集脚本**：`run_mixed.py`

**输出目录**：`stress-results/mixed/`

## 5.生产型混合（prod_mixed）

- **场景说明**：Web 服务（Nginx）+ 后台计算（Sysbench）
- **负载命令**：

```
sudo systemctl start nginx
timeout 300s ab -n100000 -c100 http://localhost/
timeout 300s sysbench --test=cpu --cpu-max-prime=20000 --num-threads=4 --time=300 run
sudo systemctl stop nginx
```

**采集脚本**：`run_prod_mixed.py`

**输出目录**：`stress-results/prod_mixed/`

### 通用采集与归因流程

1. **清理**上次采集残留文件 (`cpu.csv`, `sched.csv`)
2. **启动负载**（各场景脚本中的命令）
3. **并行启动 eBPF**：
   - 线程调度：`Data/thread/data_thread`
   - CPU 事件：   `Data/cpu/data`
4. **等待**固定时长（300s）
5. **优雅终止**：
   - 发 `SIGINT`，超时后 `terminate()` 强杀
6. **移动结果**到对应 `stress-results/<场景>/`
7. **归因分析**：
   - 按 “最近调度法” 将每条 CPU 事件归因到活跃线程
   - 输出 `attr.csv`

以上即五种场景的完整说明与脚本、目录对应关系，方便后续对比与分析。
